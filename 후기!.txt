[박세령 - 테트리스]

<소개>
- 테트리스는 누구나 살면서 한번쯤 해보았을 게임입니다.
- 혹시 사용법이 궁금하시다면 우측 상단의 물음표 아이콘을 클릭해보세요!

- 초기에 테트리스 블럭의 모양과 색을 설정해주었습니다.
- 크게 키 입력처리, 도형 생성, 도형 조작, 레벨/점수 판정, 게임종료/멈춤으로 이루어져있습니다.
- random 함수를 이용해 도형이 내려옴과 동시에 다음 도형을 보여주도록 했습니다.
- 도형이 1줄이 꽉 찼을 때 그 줄이 없어지고 위에 블럭이 내려오며 점수가 올라갑니다. (checkLine(), iSFull(), removeLine())
- 도형을 없애 점수가 올라갈수록 레벨이 올라가고, 도형이 내려오는 속도가 빨라집니다.

<소감>
- 기획부터, 디자인, 개발까지 게임 사이트를 처음부터 만든 다는 것 자체가 도전이었지만, 팀원들과 다같이 힘을 내서 좋은 결과를 얻은 것 같습니다!
- 매 회의 때마다 팀원들간의 피드백 시간, 칭찬의 시간이 너무 뜻깊었습니다.
- 서로가 잘 아는 분야를 공유하며 배우고 성장하는 프로젝트였습니다.
- 이런 기회를 가질 수 있게 해준 엘리스 고마워요 :) ♥.  


[김지환 - 룰렛]

<소개>
- 룰렛은 간단한 규칙을 가지고 있습니다. start 버튼을 눌러 룰렛을 돌리고, 룰렛이 멈추는 위치에 따라 다양한 상품을 받을 수 있습니다.


<소스코드 구조> => (이 구조는 추후에 수정될 수도 있습니다.)

- 룰렛의 자바스크립트 코드는 Roulette이라는 클래스에 모두 만들었습니다.
- 그 클래스 내부는 크게 5가지의 메소드로 구성되어 있습니다. 이 메소드들은 start 버튼을 클릭하여 이벤트가 발생했을 때 실행되는 event handler 들입니다.

- rotateRoulette 메서드를 통해 룰렛을 회전시킵니다. 
- displayResult 메서드를 이용하여 룰렛 결과 메세지를 보여줍니다.
- 결과를 보여준 이후, 다시하기 버튼을 눌렀을 때 resetRoulette 메서드와 enabledRoulette 메서드가 차례로 실행됩니다.
- resetRoulette 메서드는 시작 직전의 위치로 룰렛을 위치시킵니다. (초기화)
- enabledRoulette 메서드는 룰렛이 회전되는 동안 disabled 되었던 start 버튼을 재활성화시켜, 룰렛을 다시 즐길 수 있도록 합니다.

<소감>
- 엘리스 교육과정에서 배웠던 부분들을 최대한 모두 적용하려고 노력하였습니다.
- Javascript 파일에서는, 비동기 함수와 async/await를 이용하여 메서드를 구현하였고, class 내부에 이 메서드들을 구현하는 경험을 하며 재사용성의 필요성에 대해서도 배울 수 있었습니다.
- 또한, html, css의 프론트엔드 부분을 수정하고 개선하면서 UI 디자인적인 부분도 쉽지 않구나라는 점을 느꼈습니다.
- 협업을 하는 중간중간 디스코드 채널과 엘리스 회의 공간을 통해 토론하고, 깃허브로 커밋 관리를 하면서 협업하는 방법을 배웠습니다.
- 시간적인 여유가 많지 않아, 작은 화면이나 모바일 창에서의 반응형 처리에 아쉬움이 많은데, 프로젝트 기간이 끝나더라도 개인적인 시간을 내어서 완성할 예정입니다.


[강예정 - 가위바위포켓몬]

<소개>
- 가위바위보에서 승리하면 포켓몬 스티커 한장을 주는 게임을 구현했습니다.
- computer가 내는 걸 회전하는 부분은 setInterval로 구현했습니다.
- 가위바위보는 if문을 쓰기보다는 객체로 구현하여 코드 가독성을 늘렸고 objUser[pcHand] === (1: 승리 0: 무승부 -1: 패배)로 정의해서 간결하게 돌아가도록 했습니다.
- 플레이 전 포켓몬 도감보기라는 버튼을 클릭하시면 포켓몬들이 나오는데 아직 획득하지 못해서 검게 칠해져 있습니다. filter를 이용해 css로 검게 만들었습니다.
- 버튼에 hover시 효과음이 재생되는 기능을 넣었고 이기시면 포켓몬을 획득했다는 문구와 함께 랜덤으로 포켓몬을 얻을 수 있는 함수를 구현했습니다.
- 얻은 후에 도감을 클릭하시면 얻은 포켓몬이 표시되도록 연동했습니다.
- 또 우측 상단에 몇개의 포켓몬을 획득했는지 볼 수 있으며 151개를 모으는게 게임의 최종 목표입니다.


<소감>
- 포켓몬 빵대란에서 포켓몬빵을 하나도 얻지못해서 구상한 프로젝트입니다. 
  그래도 프로젝트를 진행하며 서러움을 풀 수 있었고 즐거웠습니다. ^0^
- 앨리스에서 배운개념들이 많이 도움이 되었고 결과물이 나왔다는게 정말 기쁩니다.
- 혼자였다면 하지 못했을 것 같아요. 팀원들이 잘 도와주고 함께 으쌰으쌰해서 성공할 수 있었습니다!
- 다음에는 백엔드적인 개념도 적용해보고 싶어요!
- 각자의 자리에서 열심히 해주신 팀원분들께 정말 감사드립니다! 나중에 또 만나요 :)

[배은지 - 제비뽑기]

<소개>
- 사람 수와 폭탄 수를 설정할 때마다 모자 그림이 생성되도록 설정했습니다.
- 사람 수보다 폭탄 수가 많아지는 오류를 잡기 위해서 사람 수를 설정하면 자동으로 폭탄 수가 (사람수-1)이 되도록 설정했습니다.

- random 합수를 사용해서 폭탄을 추첨했습니다. 중복 숫자를 제거하기 위해 while 문을 사용해서 확인했습니다.
- 결과가 나오고 2.5초 후, 결과가 출력되는 모들창이 발생합니다.
- replay 버튼을 통해 페이지를 reload하고 게임을 다시 시작할 수 있습니다.


<소감>
- 엘리스에서의 첫 프로젝트를 열정적이고 능력 있는 팀원들과 함께해서 좋았습니다.
- 프로젝트를 통해 엘리스 수업에서 배운 html, css, javascript에 대한 이해를 높일 수 있었습니다.

[김다윤 - 초성게임]

<소개>
- 주어진 힌트와 초성을 보고, 한 문제 당 5초라는 시간 내에 답안을 작성하여 진행하는 게임입니다. 문제는 총 10문제이며, 백점을 맞을 시 특별한 효과를 볼 수 있습니다.
- 게임 시작 버튼을 누르면 모든 실행 요소가 들어가있는 startGame 함수를 실행합니다.  
- quiz라는 변수에 각각 hint, initial, answer이 들어간 객체들을 집어넣고, random함수를 사용하여 중복 없이 총 10개의 숫자를 countArr의 배열에 무작위로 넣어줍니다.
  답안 제출 혹은 타이머가 다 돌아가면 count 변수를 1씩 증가시키고, countArr의 count와 같은 index에 해당되는 값(countArr[count])의 hint, initial, answer값을 정해진 태그에 innerHTML로 노출시켜 자동으로 다음 문제로 넘어가도록 구현했습니다.  
- 팀원분의 피드백을 통해 focus 함수를 사용하여 input내에 자동으로 포커스가 되도록 하고, ime-mode를 사용하여 처음 페이지 로드 시 한글로 자동 전환되도록 하였습니다.
- 시간 내에 정답을 제출한 경우, input 하단에 파란 글씨로 `'앨리스' 정답!` 이라는 문구가 노출되고 score 10점이 부여됩니다.
시간 내에 오답을 제출하거나 답을 제출하지 못한 경우에는 빨간 글씨로 정답을 보여줍니다.
정답 문구가 노출되는 1초 동안은 input과 submit button을 비활성화 시켜 enter키를 계속 눌러도 순식간에 여러개의 문제가 스킵되는 이슈를 방지했습니다.
- 10개의 문제를 모두 클리어하면 결과 모달창이 뜹니다. 얻은 점수에 따라 노출되는 문구가 각각 다르며, 백점을 맞을 시 꽃가루가 날리는 함수가 실행됩니다.

<소감> 
- 첫 프로젝트인 만큼 완성도있는 결과물을 만들고싶었고, 생각한대로 구현이 잘 되어 좋았습니다.
- 팀원들과의 끊임없는 소통과 피드백을 통해 많은 것을 배울 수 있었습니다.
- 게임 구현에만 신경을 쓰느라 앨리스 수강을 통해 배운 기술들을 많이 적용하지 못해 아쉬웠습니다. 다음 프로젝트에서는 await, async, class 등 여러 기술들을 사용해보고 싶습니다.
